  1.  class AssociationRule extends Rule

    A concrete Rule implementation to represent associations in the native unlinked model.

    A concrete Rule implementation to represent associations in the native unlinked model.

    String format: "ID:ASSOCIATION_NAME:TARGET_NAME:MULTIPLICITY" <P> where ID is the unique technical identifier of the Rule

    where ASSOCIATION_NAME is the arbitrary name of the relation

    where TARGET_NAME is the name of the ModelElement type that can be associated <P> where MULTIPLICITY is ["*" | "u_int" | "u_int...u_int" | "u_int...*"]

    See also

    Rule RuleData

  2.  class AttributeRule extends Rule

    A concrete Rule implementation to represent attributes in the native unlinked model.

    A concrete Rule implementation to represent attributes in the native unlinked model.

    String format: "ID:NAME:DATATYPE:NON_EMPTY" <P> where ID is the unique technical identifier of the Rule

    where NAME is the arbitrary name of the attribute

    where DATATYPE is the type-value of the attribute ["STRING", "NUMBER", "DATETIME"] <P> where NON_EMPTY describes if a concrete attribute must always have a non-empty value

    See also

    Rule RuleData

  3.  class ConnectionInterface extends AnyRef

    The ConnectionInterface describes the set of Slots representing the possible variants of an association target.

    The ConnectionInterface describes the set of Slots representing the possible variants of an association target.

    Each slots describes a variantTime with prefix "=", "<" and ">". A ConnectionInterface holds true for an associations, if one of the slots allows the connection by an "OR" operation. If no prefix is specified, it is treated as an "=" prefix.

  4.  class ParentRelationRule extends Rule

    A concrete Rule implementation to represent parentRelations in the native unlinked model.

    A concrete Rule implementation to represent parentRelations in the native unlinked model. This Rule does not logically distinguish between reference parentRelations and instantiated parentRelations.

    String format: "ID:PARENT_IDENTITY:PARENT_NAME" <P> where ID is the unique technical identifier of the Rule

    where PARENT_IDENTITY is the identity value of the parent ModelElement

    where TARGET_NAME is the name of the parent ModelElement
