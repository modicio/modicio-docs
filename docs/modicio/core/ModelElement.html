Companion object ModelElement

class ModelElement extends AnyRef

The ModelElement is the central class of the CoDI type-hierarchy. It forms the component of the type-composite tree.

The ModelElement contains all concrete functionality to handle associations, rules and parentRelations.

Apart from the constructor arguments, the ModelElement possess a Definition, a Registry, a DefinitionVerifier and a ModelVerifier.Those classes are not always required but should be set directly after object construction using their respective set-methods.

  41.  def unfold(): Future[Any]

    Unfolds all associations of the ModelElement.

    Unfolds all associations of the ModelElement.

    This call is not propagated to the associated ModelElements, they remain folded!

    By default, this operation is used by the TypeHandle only.

    A Registry must be available to this ModelElement because not available data is lazily reloaded.

    Note that only reference modelElements are resolved. In an instantiated case (identity != #), the reference specifications must not match with different instance identities so those should always be fetched via the respective InstanceHandle.

    returns

    Future[Unit] - after the operation terminates. The ModelElement should not be accessed before this operation terminates. Exception in error cases.
