    A concrete Rule implementation to represent associations in the native unlinked model.

    A concrete Rule implementation to represent associations in the native unlinked model.

    String format: "ID:ASSOCIATION_NAME:TARGET_NAME:MULTIPLICITY" <P> where ID is the unique technical identifier of the Rule

    where ASSOCIATION_NAME is the arbitrary name of the relation

    where TARGET_NAME is the name of the Fragment type that can be associated <P> where MULTIPLICITY is ["*" | "u_int" | "u_int...u_int" | "u_int...*"]

  2.  class AttributeRule extends Rule

    A concrete Rule implementation to represent attributes in the native unlinked model.

    A concrete Rule implementation to represent attributes in the native unlinked model.

    String format: "ID:NAME:DATATYPE:NON_EMPTY" <P> where ID is the unique technical identifier of the Rule

    where NAME is the arbitrary name of the attribute

    where DATATYPE is the type-value of the attribute ["STRING", "NUMBER", "DATETIME"] <P> where NON_EMPTY describes if a concrete attribute must always have a non-empty value
