  4.  case class FragmentData(name: String, identity: String, isTemplate: Boolean) extends Product with Serializable

    Tuple to represent a Fragment for serialisation.

    Tuple to represent a Fragment for serialisation.

    This class is produced by the Fragment itself.

    This class represents only Nodes in the reference case, but Nodes and concrete BaseModels in the instantiated case. This is because BaseModels follow their static implementation in code in the reference case (immutable). Nodes are in this case dynamic which is indicated by the reference identity. After forking a Fragment, BaseModels and Nodes are represented in data equally and immutable by this tuple.

    Name and identity form a unique combination which serves as ID.

    name

    the name of the type-fragment

    identity

    identity value

    isTemplate

    flag if the Fragment describes a template-type or instantiable-type
